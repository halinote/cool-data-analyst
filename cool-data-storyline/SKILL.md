# Visualization and Storyline

Open with: `지금은 시각화와 스토리라인만 정리합니다. 수치가 왜 결론으로 이어지는지 보여주겠습니다.` Read [references/top-line-visual-storyline.md](references/top-line-visual-storyline.md) and [references/analysis-traceability.md](references/analysis-traceability.md) completely.

Use only validated inputs with a metric contract and calculation trace. Start with a supported top line: decision answer + magnitude/evidence + driver + recommended action + caveat. Sequence 3–5 claim-led exhibits: outcome → location → driver → decision evidence → action.

Before rendering, ask the user to choose the visual style and delivery surface when they matter: `임원용 간결`, `분석가용 상세`, or `운영 모니터링`; and `정적 PPT`, `동적 웹사이트`, or `둘 다`. Recommend a choice in one sentence.

Choose a number, table, or chart based on the cognitive comparison required. Do not use a chart merely because data exists. For every chart, publish a chart contract: exact x/y axis field or formula, units, population, period, filters, series, label logic, sort order, reference lines, sample size, and plotted-value formula. Put x/y titles and units on the chart itself. Remove redundant labels that repeat titles, axes, direct labels, or legends.

Place a visible **수치 해석** block directly below each chart conclusion: displayed value → comparison/delta → driver → decision. In PPT, add a small analyst-facing methodology footer to every analytical slide: source, calculation, population, period, filters, and limitations. These two notes have different jobs; include both. Do not plot anything that cannot be traced to approved calculations.
